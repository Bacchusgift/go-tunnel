为什么选择 go-tunnel

简单、轻量、无依赖

📦

单文件部署

编译产物只有一个二进制文件，无运行时依赖，拷贝即用。Docker 镜像仅 ~10MB。

⚡

开箱即用

无需注册账号，无需配置鉴权。输入端口，回车，隧道就建立了。

🔄

交互式管理

菜单式操作界面，支持同时管理多条隧道，创建和关闭都只需选个数字。

🔒

WebSocket 传输

基于 WebSocket 双向通信，支持反向代理 HTTP/HTTPS 流量，兼容标准 Nginx。

🐳

Docker 一键部署

服务端提供 Docker Compose 配置，deploy 脚本自动拉取、构建、启动。

💻

跨平台

支持 Linux、macOS (Apple Silicon)、Windows。GitHub Actions 自动构建发布。

架构

简洁的数据通路，一行搞定

🌐 用户请求 → Nginx → go-tunnel 服务端 → WebSocket 隧道 → localhost:{port}

快速开始

三步完成部署和使用

1

部署服务端

# 克隆仓库
git clone git@github.com:Bacchusgift/go-tunnel.git
cd go-tunnel

# 配置域名
cp .env.example .env
# 编辑 TUNNEL_DOMAIN

# 启动
./deploy
2

下载客户端

# macOS / Linux / Windows
go install github.com/Bacchusgift/go-tunnel/cmd/client@latest

# 或从 Release 页下载
github.com/Bacchusgift/go-tunnel/releases
3

创建隧道

$ ./tunnel-client

🌐 服务器地址: http://tunnel.example.com

━━━ go-tunnel ━━━
  [1] 创建隧道
  [2] 关闭隧道
  [0] 退出

选择: 1
